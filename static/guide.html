使用指南

快速上手 QingChen Mail 系统

1. 配置发送通道

首先，您需要配置一个 SMTP 发送通道。QingChen Mail 支持两种模式：

  • Direct Send (内置): 直接通过本机 25 端口投递，适合有独立 IP 的服务器。
  • SMTP 中继: 使用 Gmail, Outlook, AWS SES, 阿里云 DM 等第三方服务（推荐）。
前往配置通道 →

2. 添加发信域名

为了提高送达率，您需要配置 SPF, DKIM 和 DMARC 记录。

  • 在“域名管理”中添加您的域名（如 example.com）。
  • 根据生成的 DNS 记录，前往您的域名服务商（Cloudflare, GoDaddy 等）添加 TXT 记录。
  • 点击“验证”按钮确认配置生效。
前往管理域名 →

3. 管理联系人

建立您的客户数据库。

  • 创建分组（如“订阅用户”）。
  • 批量导入联系人 (CSV/文本)。
前往联系人 →

4. 创建邮件模板

设计精美的营销邮件。

  • 支持 HTML 源码编辑。
  • 支持变量替换，例如 {name}。
前往邮件模板 →

5. 创建营销任务

选择受众和模板，开始您的营销活动。系统将自动处理队列和发送。

创建任务

常见问题

邮件进入垃圾箱怎么办？

1. 检查 SPF/DKIM 是否验证通过。
2. 避免在邮件主题中使用过多感叹号或敏感词（如“免费”、“中奖”）。
3. 尽量使用信誉良好的 SMTP 中继服务。

Direct Send 发送失败？

许多云服务器（如阿里云 ECS、腾讯云 CVM）默认封禁了出方向的 25 端口。如果是这种情况，您必须使用 SMTP 中继服务（使用 465 或 587 端口）。

如何接收回复邮件？

要接收邮件，您需要：
1. 在 DNS 中添加指向服务器公网 IP 的 MX 记录。
2. 确保防火墙开放了 25 端口。
3. 如果 QingChen Mail 监听非 25 端口（如 2525），请使用 iptables 将 25 端口流量转发到 2525。
4. 在“域名管理”中添加该域名的转发规则。

STARTTLS 是什么？

STARTTLS 允许将不安全的明文连接升级为加密连接 (SSL/TLS)。
启用它可以防止邮件内容被窃听，并提高对 Gmail 等服务商的投递信誉。
您需要在“系统设置”中配置有效的 SSL 证书才能启用此功能。

进阶配置：端口转发

如果无法直接监听 25 端口，可以使用以下方法将 25 端口流量转发到 2525（QingChen Mail 监听端口）。

iptables (Linux)

iptables -t nat -A PREROUTING -p tcp --dport 25 -j REDIRECT --to-port 2525

Nginx Stream (需安装 stream 模块)

stream {
    server {
        listen 25;
        proxy_pass 127.0.0.1:2525;
    }
}
